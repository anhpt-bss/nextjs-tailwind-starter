# What are the most common frontend security risks in 2025 and how do you mitigate them?

Security is critical for modern web apps. Interviewers want to know if you can identify and mitigate common risks.

## Why Interviewers Ask This Question

- To check your knowledge of security best practices
- To see if you can protect apps from common threats

## Common Risks (2025)

- XSS (Cross-Site Scripting)
- CSRF (Cross-Site Request Forgery)
- Supply chain attacks
- Insecure dependencies
- Sensitive data exposure

## Mitigation Strategies

- Sanitize user input
- Use Content Security Policy (CSP)
- Keep dependencies up to date
- Use HTTPS and secure cookies
- Implement proper authentication/authorization

## Example

```js
// Sanitize input
const safe = DOMPurify.sanitize(userInput)
```

## Interview Tips

- Mention OWASP Top 10
- Discuss real-world incidents

## Useful English Phrases & Sentence Starters

- The most common risks are ...
- I mitigate them by ...
- For example, ...

## Sample Answer

> The most common frontend security risks in 2025 are XSS, CSRF, and supply chain attacks. I mitigate them by sanitizing input, using CSP, keeping dependencies updated, and following security best practices.
