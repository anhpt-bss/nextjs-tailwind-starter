# What is static code analysis and linting?

Static code analysis and linting help maintain code quality. Interviewers want to know if you use these tools to catch bugs and enforce standards.

## Why Interviewers Ask This Question

- To check your knowledge of code quality tools
- To see if you can enforce best practices

## Static Code Analysis

- Analyzes code without running it
- Finds bugs, security issues, code smells

## Linting

- Checks code style and formatting
- Enforces coding standards (ESLint, Prettier)

## Example

```js
// ESLint config
{
  "rules": {
    "no-unused-vars": "error"
  }
}
```

## Interview Tips

- Mention tools (ESLint, SonarQube, Prettier)
- Discuss integration with CI/CD

## Useful English Phrases & Sentence Starters

- I use static analysis to ...
- Linting helps ...
- For example, ...

## Sample Answer

> I use static code analysis tools like ESLint and SonarQube to catch bugs and enforce coding standards. Linting helps keep the codebase clean and consistent, and I integrate these tools into CI/CD pipelines.
