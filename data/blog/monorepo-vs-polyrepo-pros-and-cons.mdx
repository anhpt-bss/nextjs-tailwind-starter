# Monorepo vs Polyrepo: pros and cons?

Choosing between monorepo and polyrepo is an important architectural decision. Interviewers want to know if you understand the trade-offs.

## Why Interviewers Ask This Question

- To check your experience with project organization
- To see if you can justify your choices

## Monorepo

- Single repository for multiple projects/packages
- Easier code sharing, unified tooling, atomic commits
- Can become complex as it grows

## Polyrepo

- Separate repositories for each project/package
- Simpler for small teams, clear boundaries
- Harder to share code and manage dependencies

## Interview Tips

- Compare collaboration, tooling, scalability
- Give real-world examples

## Useful English Phrases & Sentence Starters

- Monorepo is ...
- Polyrepo is ...
- I prefer ... when ...

## Sample Answer

> Monorepo is a single repository for multiple projects, making code sharing and tooling easier, but can become complex at scale. Polyrepo uses separate repositories, which is simpler for small teams but harder to manage shared code. I choose based on team size and project needs.
